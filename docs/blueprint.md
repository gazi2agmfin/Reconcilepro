# **App Name**: ReconcilePro

## Core Features:

- Data Entry Form: Web form mimicking the provided bank reconciliation format, allowing users to input data for each field: Bank Name, Bank Code, Reconciliation Date, Reconciliation Month, Balance as per Bank/Book, additions and deductions as shown on the provided document. Reconciliation Date and Month are fixed per user until changed.
- Automated Bank Name Population: Automatically populate the Bank Name field based on the Bank Code entered by the user.
- Automated Calculations: Automatically calculate total additions, total deductions, and the corrected balance, with an indication of any differences.
- Data Persistence: Store the entered data for each reconciliation statement. Save, load, add, and edit previously saved reconciliations.
- User Authentication: Secure user login and authentication to protect data and restrict access.
- Data Search and Retrieval: Enable users to search for specific reconciliation statements to reprint or review.
- Dashboard Analytics: Display a dashboard providing a monthly overview of reconciliation numbers and key metrics.
- Admin Panel: Provide an admin interface to view, edit, and manage all user data and accounts.
- Field Narration: Allow users to input textual narration for each data field to provide additional context or explanations between the figures.

## Style Guidelines:

- Primary color: Deep Indigo (#3F51B5) for trustworthiness and stability.
- Background color: Very light gray (#F0F2F5), near-white to give a clean look without harshness.
- Accent color: Amber (#FFC107) to highlight key interactive elements and important data fields.
- Body font: 'Inter', sans-serif, for clear and legible display of data and text.
- Headline font: 'PT Sans', sans-serif, providing a slightly warmer, but still professional, tone to titles.
- Use clean, crisp, and professional icons from a consistent set, representing actions such as 'add,' 'edit,' 'search,' and 'print.'
- Mimic the structure of the original bank reconciliation document, maintaining a clear and logical flow with distinct sections for each data category.
- Subtle transitions and animations for form interactions, such as saving or loading data, to provide feedback without being distracting.
